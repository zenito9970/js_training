onclick がダメで addEventListener のほうがよいってやつどういうこと？？？ってなったので検証．

表示 (1) と (2) は class: "sayButton" をもつボタンに， addEventListener を使って関数を設定している．
(1) のボタンには HTML 側で onclick="say1()" を設定しており，ページの読み込みが終わると同時に addEventListener でイベントが追加されるので， say1() が2回呼び出されることになる．

表示 (3) と (4) は js 側で onclick に代入してみたやつ．
(3) は HTML で onclick="say1()" を設定しているけど， js 側で代入しているので上書きされてしまっている．

これ個人で1ページ分だけやる分にはあんまり変わらんのでは？？という感じもするんだけどどうなんやろ．
たぶん，ライブラリを作るときに onclick に上書きすると他のやつに迷惑かけるからやめよう！みたいな話なんじゃないかなあ……？
ページが動的に変更されうる場合の window.onload の挙動がよくわからないので，そのへんを検証すべきかもしれない．
ページを自動生成するときなんかは， HTML 側で onclick を設定するほうが作りやすいよなあと思ってしまう．
